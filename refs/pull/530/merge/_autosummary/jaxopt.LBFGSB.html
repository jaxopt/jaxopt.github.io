L-BFGS-B solver.

L-BFGS-B is a version of L-BFGS that incorporates box constraints on variables.

funÔÉÅ

a smooth function of the form fun(x, *args, **kwargs).

value_and_gradÔÉÅ

whether fun just returns the value (False) or both the value and gradient (True). See base.make_funs_with_aux for details.

has_auxÔÉÅ

whether fun outputs auxiliary data or not. If has_aux is False, fun is expected to be scalar-valued. If has_aux is True, then we have one of the following two cases. If value_and_grad is False, the output should be value, aux = fun(...). If value_and_grad == True, the output should be (value, aux), grad = fun(...). At each iteration of the algorithm, the auxiliary outputs are stored in state.aux.

maxiterÔÉÅ

maximum number of proximal gradient descent iterations.

tolÔÉÅ

tolerance of the stopping criterion.

stepsizeÔÉÅ

a stepsize to use (if <= 0, use backtracking line search), or a callable specifying the positive stepsize to use at each iteration.

linesearch_initÔÉÅ

strategy for line-search initialization. By default, it will use ‚Äúincrease‚Äù, which will increased the step-size by a factor of increase_factor at each iteration if the step-size is larger than min_stepsize, and set it to max_stepsize otherwise. Other choices are ‚Äúmax‚Äù, that initializes the step-size to max_stepsize at every iteration, and ‚Äúcurrent‚Äù, that uses the step-size from the previous iteration.

stop_if_linesearch_failsÔÉÅ

whether to stop iterations if the line search fails. When True, this matches the behavior of core JAX.

conditionÔÉÅ

Deprecated. Condition used to select the stepsize when using backtracking linesearch

maxlsÔÉÅ

maximum number of iterations to use in the line search.

decrease_factorÔÉÅ

Deprecated. factor by which to decrease the stepsize during backtracking line search (default: 0.8).

increase_factorÔÉÅ

factor by which to increase the stepsize during line search (default: 1.5).

max_stepsizeÔÉÅ

upper bound on stepsize.

min_stepsizeÔÉÅ

lower bound on stepsize.

history_sizeÔÉÅ

size of the memory to use.

use_gammaÔÉÅ

whether to initialize the Hessian approximation with gamma * theta, where gamma is chosen following equation (7.20) of ‚ÄòNumerical Optimization‚Äô [2]. If use_gamma is set to False, theta is used as initialization.

implicit_diffÔÉÅ

whether to enable implicit diff or autodiff of unrolled iterations.

implicit_diff_solveÔÉÅ

the linear system solver to use.

jitÔÉÅ

whether to JIT-compile the optimization loop (default: True).

unrollÔÉÅ

whether to unroll the optimization loop (default: ‚Äúauto‚Äù).

verboseÔÉÅ

whether to print error on every iteration or not. Warning: verbose=True will automatically disable jit.
